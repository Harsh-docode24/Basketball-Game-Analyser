# Documentation Style Guide

## Writing Principles

- Use active voice.
- Keep sentences concise.
- Avoid ambiguous terminology.
- Explain deterministic logic clearly.
- Separate conceptual and procedural information.

## Structure Principles

- Concept: Explain what something is.
- Task: Explain how to perform an action.
- Reference: Provide technical details.

## Formatting

- Use hierarchical headings.
- Maintain consistent terminology.
- Keep installation steps reproducible.
